Executive Summary

Selected Projects

Project 01 · AXE

RL Order-Execution Engine

AXE는 parent order를 시간·수량·시장 제약 안에서 child-order actions로 나눠 집행하는 reinforcement-learning execution engine입니다. 프로젝트의 핵심은 policy 하나가 아니라 simulator, action/reward interface, critic modeling, release gate가 같은 루프로 맞물리는 점입니다. 긴 horizon의 task를 관측·행동·보상·검증 단위로 쪼개고 다시 운영 기준으로 닫는 decision-system 문제입니다.

Project 02 · GGTDE

Critic Uncertainty Extension

GGTDE는 AXE의 component-critic work 이후 남은 critic uncertainty를 별도 연구 질문으로 떼어낸 프로젝트입니다. TD error를 Gaussian noise로만 보지 않고 Generalized Gaussian으로 모델링해 tail thickness와 aleatoric uncertainty를 critic loss 안에서 다룹니다.

Project 03 · FAN-BEATS

Feature-Aligned N-BEATS

FAN-BEATS는 N-BEATS/N-HiTS 계열의 stack-wise feature distribution을 Sinkhorn divergence로 정렬해 unseen domain forecasting을 다룬 domain-shift 연구입니다. domain과 데이터 조건이 달라져도 유지되는 representation을 찾는 문제라는 점에서 같은 문제의식으로 이어집니다.

Project 01 · AXE · Problem Setting

Long-Horizon Execution Under Model and Market Constraints

Why Terminal Reward Failed

order execution duration은 길고, market dynamics는 모델 크기에 비해 훨씬 복잡했습니다. on-prem deployment와 latency constraint 때문에 모델을 크게 키우기도 어려웠고, 대략 <3M parameters 안에서 policy를 안정화해야 했습니다. 마지막 performance 하나만 reward로 주면 긴 horizon에서 convergence가 잘 나오지 않았습니다.

Action-Type Reward

그래서 order, cancel, replace, maintain처럼 실제 집행 과정에서 발생하는 action type별로 reward 기준을 나눴습니다. policy가 단순히 “마지막 성과가 좋았는지”가 아니라 각 component가 execution objective에 어떻게 기여했는지 학습하도록 만든 것입니다.

Credit Assignment Pressure

reward component가 많아지자 overall reward fluctuation이 복잡해졌고, execution performance가 중요하다 보니 order-execution time difference 신호가 다른 reward를 덮는 문제가 커졌습니다. 이 지점에서 component reward별 critic modeling과 credit assignment가 필요해졌습니다.

Core Problem: 작은 모델로 긴 execution horizon을 다루려면, final metric 하나가 아니라 action component별 reward와 critic이 필요했습니다.

Project 01 · AXE · Model Architecture

AXE Model Architecture

Encoder

visible market state와 execution job context를 compact representation으로 압축합니다. book/trade feature와 remaining quantity/time, pending orders를 같은 decision state 안에서 다룹니다.

Model Torso

small on-prem model budget 안에서 policy와 value가 필요한 representation을 분리할 수 있게 설계했습니다. shared torso뿐 아니라 separate-torso PPG도 이 가설에서 실험했습니다.

Action and Reward

policy head는 action type, quantity ratio, price level을 내고, rewarder는 execution quality를 component rewards로 나눕니다. critic loss는 이 reward decomposition을 다시 model update로 돌려보냅니다.

Project 01 · AXE · Credit Assignment

Multi-Head Critic for Component Rewards

Problem

component reward가 많아지자 전체 reward fluctuation이 커졌고, 특히 order-execution time difference처럼 execution performance와 직접 연결된 신호가 다른 component를 쉽게 덮었습니다.

Method

reward component마다 critic/value head를 두고 component별 advantage를 계산했습니다. gradient는 importance weighting과 PCGrad로 조정해, 강한 component 하나가 나머지 학습 신호를 지워버리지 않게 했습니다.

Outcome

목표는 평균 성능 하나를 올리는 것이 아니라 performance variance를 낮추고 downside CVaR을 올리는 것이었습니다. 개인 연구 방향에서 출발했지만, 이후 팀의 학습·검증 workflow 일부로 재사용되었습니다.

Project 02 · GGTDE · Critic Uncertainty Extension

TD Error Tailedness as Aleatoric Uncertainty

p(δ)=β2αΓ(1/β)exp(−(|δ|α)β)shape β
= tail thickness
σ2(β)=α2Γ(3/β)Γ(1/β)aleatoric variance
closed form

β=2는 Gaussian(MSE), β=1은 Laplace(MAE)입니다. β를 학습하면 TD error의 tail thickness와 kurtosis를 critic loss 안에서 다룰 수 있습니다.

Research Trigger

MHC 이후에도 residual randomness가 남았습니다. 이 문제를 AXE 내부 튜닝으로만 보지 않고, critic uncertainty를 별도 연구 질문으로 떼어냈습니다. reward뿐 아니라 TD error 자체의 tailedness를 다루면 intrinsic randomness를 critic loss 안에서 설명할 수 있겠다는 아이디어였습니다.

Modeling Choice

TD error likelihood를 Generalized Gaussian으로 두고 shape β를 학습합니다. β는 tail thickness를 조절하므로, Gaussian/MSE가 놓치는 sharp peak와 heavy tail을 value estimation uncertainty의 일부로 표현할 수 있습니다.

Why It Matters

실제 decision problem에서는 평균적으로 작은 value error보다 드물지만 큰 error가 release risk를 만듭니다. GGTDE는 그런 rare-but-costly critic error를 별도 신호로 다루려는 연구였고, 제 domain-shift 관심이 RL 쪽으로 이어진 예입니다.

Project 01 · AXE · Curriculum and Representation

Curriculum Learning for Execution Jobs

Difficulty Axes

difficulty는 market condition, liquidity, side/session 같은 trading context와 urgency, remaining quantity/time 같은 target basis, 그리고 state/policy complexity와 reward schedule 같은 training state로 나눠 봤습니다.

Curriculum Move

easy environments에서 calibration한 뒤 abnormal cases로 넓히는 LP(linear-programming)-optimal performance curriculum과, 어려운 환경까지 uniform sampling한 뒤 실제 distribution에 맞추는 target-basis curriculum을 비교했습니다. policy/value representation 분리 가설은 separate-torso PPG로도 확인했습니다.

Observed Pattern

LP-optimal performance curriculum은 downside CVaR 개선에, target-basis curriculum은 median performance 개선에 유리했습니다. 두 축을 함께 쓰는 방식이 이후 training workflow에 반영되었습니다.

Interpretation: favorable market에서는 안정적인 execution을, 작은 target basis에서는 timing의 중요도를 더 크게 반영한 것으로 해석하고 있습니다. 다만 이 해석은 아직 가설 수준이며, 별도 검증 연구로 마무리되지는 않았습니다.

Project 01 · AXE · Simulation and Production Evaluation

One Evaluation Loop From Simulator to Release

Market Simulator

market simulator는 candidate policy를 live 적용 전에 먼저 검증하는 시장 환경입니다. replayed market/job distribution에서 fill, slippage, completion behavior를 확인해 sim-live gap을 관리합니다.

Evaluation Slices

같은 candidate를 market, cap size, category, liquidity, side, session처럼 비슷한 의미를 갖는 group으로 나눠 봅니다. avg/std/median과 함께 CVaR, min performance, win rate도 함께 봅니다.

Release Feedback

tail이나 특정 regime에서 깨지는 candidate는 release하지 않습니다. 정기 evaluation과 stakeholder feedback은 다음 simulator/evaluation/reward task로 되돌아갑니다.

Loop: simulator는 evaluation이 확인한 failure mode를 재현하고, evaluation은 다음 simulator gap과 reward fix로 이어집니다. 이렇게 simulation fidelity와 production evaluation이 하나의 release loop로 닫힙니다.

Project 03 · FAN-BEATS · Domain-Shift Forecasting

Domain-Shift Representation Learning

Original Question

석사 과정에서 가장 오래 붙잡은 질문은 domain이 바뀌어도 유지되는 feature representation을 어떻게 만들 수 있는가였습니다. FAN-BEATS는 그 질문을 time-series forecasting setting에서 다룬 연구입니다.

Method

각 stack의 feature measure를 simplex 위로 올리고, source domain pair 중 가장 안 맞는 쌍의 Sinkhorn divergence를 줄이는 minimax alignment를 사용했습니다. 성능 향상보다 중요한 것은 unseen target domain에서 feature가 무너지지 않게 하는 구조였습니다.

Through-Line

FAN-BEATS는 representation learning, AXE는 reinforcement learning이지만 중심 질문은 같습니다. domain · data · distribution shift 아래에서 모델이 어떤 feature와 decision rule을 유지해야 하는가입니다.

Closing

One Through-Line Across the Work

FAN-BEATS, AXE, GGTDE는 서로 다른 문제처럼 보이지만, 제게는 하나의 질문으로 묶입니다. domain · data · distribution shift 아래에서 representation, policy, critic, evaluation이 함께 버티는 구조를 어떻게 만들 것인가.

Representation

FAN-BEATS에서는 domain이 바뀌어도 유지되는 feature representation을 보았습니다. 모델이 어떤 신호를 버리고 어떤 신호를 남겨야 하는지가 출발점이었습니다.

Decision

AXE에서는 long-horizon execution을 action, reward, critic interface로 나누고 다시 simulator와 release gate로 닫았습니다. policy 하나가 아니라 운영 가능한 decision loop가 핵심이었습니다.

Evaluation

GGTDE와 production evaluation에서는 평균 성능보다 tail, regime, uncertainty, release boundary를 봤습니다. 실제 환경에 올릴 수 있는 범위만 주장하는 것이 마지막 기준입니다.
